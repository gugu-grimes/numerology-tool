# 数据库设计 — 00. 通用与外壳模块

## 概述

本模块为应用级 UX 偏好与外壳能力提供基础设施，涵盖术语悬浮提示、多语言扩展基础和移动端适配三大子模块。

根据架构契约 `architecture.md §③ 数据归属` 的明确规定：**模块 00（通用与外壳）不持有持久化数据**。本模块的所有数据均在前端运行时管理，不涉及后端数据库表的创建与持久化存储：

- **术语定义数据**：来源于 `0.common/glossary.md`，在前端构建时打包为静态 JSON（`code/frontend/src/i18n/locales/` 或 `code/frontend/src/assets/`），运行时由前端组件直接引用，无需后端端点或数据库表。
- **多语言语言包**：由 vue-i18n 管理，语言包文件存储于 `code/frontend/src/i18n/locales/` 目录，为纯前端静态资源，无后端存储。
- **语言偏好与移动端布局偏好**：通过浏览器 `localStorage` 持久化（Pinia store + localStorage 插件），无后端同步需求。

## 表清单

N/A — 本模块无需数据库表。术语定义数据来自前端打包的静态 JSON；语言包为前端静态资源；用户偏好通过浏览器 localStorage 管理。

## 跨模块数据引用

| 引用数据 | 来源模块 | 引用方式 |
|----------|----------|----------|
| 术语定义（glossary） | `0.common/glossary.md` | 前端构建时打包为 JSON，运行时引用 |
| Chart 排盘结果 | 01. 八字排盘与历法模块 | 术语提示组件在排盘结果页面内嵌使用，通过 Chart ID 引用排盘上下文 |

## Prisma Schema 影响

无 — 本模块不新增 Prisma 模型，不修改已有模型。`schema.prisma` 中无需为本模块添加任何内容。
