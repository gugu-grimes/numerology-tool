# API 设计 — 00. 通用与外壳模块

## 概述

本模块为应用级 UX 基础设施，所有能力均为前端运行时行为，不涉及后端 REST API 端点。根据 `code-structure.md §5.1 通用与外壳模块` 后端部分的明确说明："术语定义数据来自 `0.common/glossary.md` 的术语表，前端直接打包术语 JSON，无独立后端端点。"多语言切换与移动端适配同样为纯前端能力。

## 1. 子模块 API 汇总

### 1.1 术语悬浮提示

N/A — 无后端 API 端点。术语定义数据在前端构建时打包为静态 JSON，由 `TermTooltip.vue` 组件运行时引用。

### 1.2 多语言扩展基础

N/A — 无后端 API 端点。语言切换由 vue-i18n 前端插件驱动，语言包为前端静态资源，语言偏好通过 localStorage 持久化。

### 1.3 移动端适配

N/A — 无后端 API 端点。响应式布局与断点检测为纯前端运行时行为，由 `AppLayout.vue` 和 `MobileNavBar.vue` 组件实现。

## 2. 前端服务接口

虽然本模块无后端 API 端点，但定义以下前端服务接口（TypeScript 类型与函数签名），供各子模块内部及跨模块调用使用。

### 2.1 术语注册服务（TermRegistryService）

| 方法 | 签名 | 说明 |
|------|------|------|
| 查询术语定义 | `lookupTerm(termKey: string): TermDefinition \| null` | 根据术语键名查找定义，返回术语名称、定义与出处；未找到返回 `null` |
| 获取全部术语 | `getAllTerms(): TermDefinition[]` | 返回术语表全量数据，用于页面内容扫描与高亮 |
| 同步术语表 | `syncFromGlossary(): void` | 从打包的 glossary JSON 同新术语表数据 |

**术语定义类型**：

```typescript
interface TermDefinition {
  termKey: string;      // 术语键名，如 "yongShen"
  name: string;          // 术语中文名，如 "用神"
  definition: string;    // 术语定义
  source: string;        // 出处标记，如 "URS §5"
}
```

### 2.2 国际化服务（I18nService / Pinia Store）

| 方法 | 签名 | 说明 |
|------|------|------|
| 切换语言 | `switchLocale(locale: string): Promise<void>` | 切换当前界面语言，加载对应语言包并更新 vue-i18n locale |
| 获取当前语言 | `getCurrentLocale(): string` | 返回当前激活的语言标识（如 `zh-CN`） |
| 保存语言偏好 | `savePreference(locale: string): void` | 将语言偏好写入 localStorage |
| 加载语言偏好 | `loadPreference(): string \| null` | 从 localStorage 读取语言偏好；无记录时返回 `null` |
| 获取语言列表 | `getLocaleList(): LocaleInfo[]` | 返回所有已安装语言的信息列表（语言标识、本地名称、翻译完成度） |
| 消息回退解析 | `resolveMessage(key: string, locale: string): ResolvedMessage` | 按 `locale → zh-CN` 回退链解析翻译键，返回翻译文本与回退标记 |

**类型定义**：

```typescript
interface LocaleInfo {
  locale: string;          // 语言标识，如 "zh-CN", "en"
  localName: string;      // 本地名称，如 "简体中文", "English"
  completionRate: number;  // 翻译完成度 0-100
}

interface ResolvedMessage {
  text: string;           // 最终显示文本
  isFallback: boolean;    // 是否为回退显示（非所选语言原文）
}
```

### 2.3 响应式布局服务（ResponsiveService）

| 方法 | 签名 | 说明 |
|------|------|------|
| 检测断点 | `detectBreakpoint(): 'mobile' \| 'desktop'` | 根据当前视口宽度与断点阈值判断布局模式 |
| 监听断点变化 | `onBreakpointChange(callback: (mode: 'mobile' \| 'desktop') => void): () => void` | 注册断点变化监听器，返回取消监听函数 |
| 获取断点阈值 | `getBreakpointValue(): number` | 返回当前断点阈值（px），默认 768 |

## 3. 错误处理

本模块无后端 API，不涉及 HTTP 错误响应。前端服务接口的错误处理策略：

| 场景 | 处理方式 |
|------|----------|
| 术语未收录 | `lookupTerm()` 返回 `null`，组件保持高亮样式但不弹出提示浮层 |
| 语言包加载失败 | `switchLocale()` 回退到 `zh-CN`（默认语言），前端 console 输出警告 |
| 语言包不完整 | `resolveMessage()` 对缺失键回退到 `zh-CN` 并标记 `isFallback: true`，UI 以灰色底色标注 |
| localStorage 不可用 | 语言偏好降级为会话级（刷新后回到默认语言 `zh-CN`），不影响核心功能 |
