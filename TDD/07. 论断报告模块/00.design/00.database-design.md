# 数据库设计 — 07. 论断报告模块

## 概述

本模块负责论断报告的章节组织与导出数据持久化。核心数据表为 `Report`（论断报告表），存储命盘对应的报告元数据、章节选择配置与汇编内容。`Report` 通过 `chartId` 引用模块 01 的 Chart 实体（排盘数据），其汇编内容来自模块 01–06 的分析输出，但本模块仅存储最终汇编结果，不重复存储各上游模块的原始分析数据。

根据架构契约 `architecture.md §③ 数据归属`：**模块 07 拥有 Report（论断报告）实体**。

ID 策略遵循 ADR-004（自增整数）；时间存储遵循 ADR-001（UTC）；软删除遵循 ADR-002（`deleted_at` 字段）。

## 表清单

| 表名 | 说明 | 归属模块 |
|------|------|----------|
| Report | 论断报告记录，存储报告元数据、章节选择配置与汇编内容 | 本模块（07） |

## 模块专属表

### Report（论断报告）

存储命盘的论断报告记录，每条记录对应一次报告生成结果。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Int | PK, AUTOINCREMENT | 报告记录 ID（ADR-004 自增整数） |
| chartId | Int | NOT NULL, FK → Chart.id | 关联命盘 |
| chapterSelection | Json | NOT NULL | 章节选择配置（见 JSON 结构定义） |
| reportContent | Json | NOT NULL | 汇编后的报告内容（见 JSON 结构定义） |
| status | String | NOT NULL | 报告状态：`"draft"` / `"confirmed"` |
| createdAt | DateTime | NOT NULL, DEFAULT now() | 创建时间 |
| updatedAt | DateTime | NOT NULL, @updatedAt | 更新时间 |
| deletedAt | DateTime? | — | 软删除时间戳（ADR-002） |

**索引**：

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_report_chart | chartId | NON-UNIQUE | 同一命盘可生成多份报告 |

**外键**：

| 字段 | 引用 | 说明 |
|------|------|------|
| chartId | Chart.id | 所属命盘，级联删除 |

#### chapterSelection JSON 结构定义

```json
{
  "chapters": ["overview", "diagnosis", "yongshen", "judgment", "dayun_yaoxiao", "shensha", "hechong", "conclusion"],
  "overviewMandatory": true
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| chapters | Array\<String\> | 是 | 选中章节的 key 列表，按主线顺序排列 |
| overviewMandatory | Boolean | 是 | 基本命盘概览章节是否强制包含（始终为 `true`，见 PRD 业务规则 2） |

**章节 key 映射表**：

| key | 章节名称 | 数据来源模块 |
|-----|----------|-------------|
| overview | 基本命盘概览 | 01-八字排盘与历法 |
| diagnosis | 命局诊断（识病） | 04-辨病与用神（病机识别部分） |
| yongshen | 用药方略（用神喜忌） | 04-辨病与用神（用神推导部分） |
| judgment | 分维论断 | 04-辨病与用神（断吉凶部分） |
| dayun_yaoxiao | 岁运药效 | 06-大运流年 |
| shensha | 神煞与特殊格局 | 05-神煞标注 |
| hechong | 合冲刑害详解 | 03-合冲刑害 |
| conclusion | 综合论断与建议 | 汇总以上各章结论 |

#### reportContent JSON 结构定义

```json
{
  "overview": {
    "title": "基本命盘概览",
    "content": "命主甲木日主，生于寅月...",
    "basisChain": []
  },
  "diagnosis": {
    "title": "命局诊断（识病）",
    "content": "日主甲木得令于寅月...",
    "basisChain": [
      {
        "disease": "日主过旺",
        "medicine": "官杀克制",
        "conclusion": "性格刚直，易与人冲突"
      }
    ]
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| {chapterKey} | Object | 是 | 各章节内容对象，key 对应 chapterSelection 中的章节 key |
| {chapterKey}.title | String | 是 | 章节标题 |
| {chapterKey}.content | String | 是 | 章节正文内容（含论断文本） |
| {chapterKey}.basisChain | Array | 是 | 病机依据链列表（PRD 业务规则 4：每条论断须关联"病→药→结论"） |

**basisChain 内嵌结构定义**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| disease | String | 是 | 所依据的病（病机描述） |
| medicine | String | 是 | 所用之药（用神/喜忌描述） |
| conclusion | String | 是 | 推导出的结论 |

## Prisma Schema

```prisma
/// 论断报告记录 — 模块 07 创建并拥有
model Report {
  id                Int       @id @default(autoincrement())
  chartId           Int                                                // 关联命盘
  chapterSelection  Json                                               // 章节选择配置
  reportContent     Json                                               // 汇编后的报告内容
  status            String                                             // 报告状态："draft" / "confirmed"
  createdAt         DateTime  @default(now())
  updatedAt         DateTime  @updatedAt
  deletedAt         DateTime?                                          // 软删除（ADR-002）

  chart             Chart     @relation(fields: [chartId], references: [id], onDelete: Cascade)

  @@index([chartId], map: "idx_report_chart")
  @@map("report_records")
}
```
