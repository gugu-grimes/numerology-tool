# 数据库设计 — 01. 八字排盘与历法模块

## 概述

本模块负责八字排盘的核心数据持久化，包含命盘（Chart）与四柱（Pillar）两张数据表。命盘记录用户的出生信息、排盘输入参数与计算结果摘要；四柱记录每柱的天干、地支、藏干与纳音五行数据。农历公历互转与节气交接点查询为纯计算功能（基于内置万年历数据库），不涉及数据持久化存储。

根据架构契约 `architecture.md §③ 数据归属`：**模块 01 创建并拥有 Chart 与 Pillar 实体，其他模块（02–08）通过 Chart ID 引用排盘数据**。

ID 策略遵循 ADR-004（自增整数）；时间存储遵循 ADR-001（UTC）；软删除遵循 ADR-002（`deleted_at` 字段）。

## 表清单

| 表名 | 说明 | 归属模块 |
|------|------|----------|
| Chart | 八字命盘记录，存储出生信息与排盘结果摘要 | 本模块（01） |
| Pillar | 四柱数据，每条记录对应一柱（年/月/日/时） | 本模块（01） |

## 模块专属表

### Chart（命盘）

存储八字排盘的输入参数与计算结果摘要。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Int | PK, AUTOINCREMENT | 命盘 ID（ADR-004 自增整数） |
| birthDate | DateTime | NOT NULL | 公历出生日期时间（UTC，ADR-001） |
| birthDateType | String | NOT NULL | 输入方式：`"solar"` / `"lunar"` |
| gender | String | NOT NULL | 性别：`"male"` / `"female"` |
| lunarBirthInfo | Json? | — | 农历出生信息（仅当 `birthDateType` 为 `"lunar"` 时存储，见 JSON 结构定义） |
| dayMaster | String | NOT NULL | 日主天干（如 `甲`、`乙`） |
| dayMasterElement | String | NOT NULL | 日主五行属性（金/木/水/火/土） |
| jieqiName | String? | — | 出生时刻对应节气名称 |
| jieqiTime | DateTime? | — | 节气交接时刻（UTC） |
| isBeforeLichun | Boolean? | — | 出生时间是否在立春之前 |
| zhourule | String | NOT NULL, DEFAULT `"early_zi"` | 子时处理方式：`"early_zi"`（早子时）/ `"late_zi"`（夜子时） |
| createdAt | DateTime | NOT NULL, DEFAULT now() | 创建时间 |
| updatedAt | DateTime | NOT NULL, @updatedAt | 更新时间 |
| deletedAt | DateTime? | — | 软删除时间戳（ADR-002，值为 `null` 表示未删除） |

**索引**：

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_chart_created | createdAt | ASC | 按创建时间排序查询，支持命盘历史列表 cursor 分页 |

**外键**：无（Chart 为顶层实体，本模块内无外键引用）

### Pillar（四柱）

存储命盘的四柱数据，每条记录对应一柱（年柱/月柱/日柱/时柱）。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Int | PK, AUTOINCREMENT | 四柱记录 ID |
| chartId | Int | NOT NULL, FK → Chart.id | 所属命盘 |
| position | String | NOT NULL | 柱位：`"year"` / `"month"` / `"day"` / `"hour"` |
| heavenlyStem | String | NOT NULL | 天干（如 `甲`、`乙`） |
| earthlyBranch | String | NOT NULL | 地支（如 `子`、`丑`） |
| hiddenStems | Json | NOT NULL | 藏干数据（见 JSON 结构定义） |
| nayin | String? | — | 纳音五行（如 `海中金`，可选显示） |
| createdAt | DateTime | NOT NULL, DEFAULT now() | 创建时间 |
| updatedAt | DateTime | NOT NULL, @updatedAt | 更新时间 |

**索引**：

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_pillar_chart_position | chartId + position | UNIQUE | 同一命盘的同一柱位唯一 |

**外键**：

| 字段 | 引用 | 说明 |
|------|------|------|
| chartId | Chart.id | 所属命盘，级联删除 |

### JSON 结构定义

#### lunarBirthInfo

当 `birthDateType` 为 `"lunar"` 时，`lunarBirthInfo` 字段存储以下 JSON 结构：

```json
{
  "year": 1990,
  "month": 5,
  "day": 23,
  "isLeapMonth": false
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| year | Int | 是 | 农历年 |
| month | Int | 是 | 农历月 |
| day | Int | 是 | 农历日 |
| isLeapMonth | Boolean | 否 | 是否闰月，默认 `false` |

#### hiddenStems

`hiddenStems` 字段存储每个地支的藏干数据：

```json
{
  "mainQi": "甲",
  "middleQi": "丙",
  "residualQi": "戊"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| mainQi | String | 是 | 本气（地支藏干中力量最强的天干） |
| middleQi | String? | 否 | 中气（力量次于本气的天干，部分地支无中气） |
| residualQi | String? | 否 | 余气（力量最弱的天干，部分地支无余气） |

示例：地支 `子` 仅有一个藏干 `癸`，则 `hiddenStems` 为：

```json
{
  "mainQi": "癸",
  "middleQi": null,
  "residualQi": null
}
```

## 跨模块数据引用

| 引用数据 | 来源模块 | 引用方式 | 说明 |
|----------|----------|----------|------|
| Chart + Pillar | 本模块（01） | 本模块自有 | 排盘核心数据 |

## Prisma Schema

```prisma
/// 八字命盘记录 — 模块 01 创建并拥有
model Chart {
  id               Int       @id @default(autoincrement())
  birthDate        DateTime                                 // 公历出生日期时间（UTC）
  birthDateType    String                                   // 输入方式："solar" | "lunar"
  gender           String                                   // 性别："male" | "female"
  lunarBirthInfo   Json?                                    // 农历出生信息（birthDateType=lunar 时存储）
  dayMaster        String                                   // 日主天干
  dayMasterElement String                                   // 日主五行属性
  jieqiName        String?                                  // 出生时刻节气名称
  jieqiTime        DateTime?                                // 节气交接时刻（UTC）
  isBeforeLichun   Boolean?                                 // 是否在立春之前
  zhourule         String    @default("early_zi")            // 子时处理方式："early_zi" | "late_zi"
  createdAt        DateTime  @default(now())
  updatedAt        DateTime  @updatedAt
  deletedAt        DateTime?                                // 软删除（ADR-002）

  pillars          Pillar[]

  @@index([createdAt], map: "idx_chart_created")
  @@map("charts")
}

/// 四柱数据 — 每条记录对应一柱（年/月/日/时）
model Pillar {
  id             Int       @id @default(autoincrement())
  chartId        Int
  position       String                                    // 柱位："year" | "month" | "day" | "hour"
  heavenlyStem   String                                    // 天干
  earthlyBranch  String                                    // 地支
  hiddenStems    Json                                      // 藏干数据：{mainQi, middleQi, residualQi}
  nayin          String?                                   // 纳音五行（可选）
  createdAt      DateTime  @default(now())
  updatedAt      DateTime  @updatedAt

  chart          Chart     @relation(fields: [chartId], references: [id], onDelete: Cascade)

  @@unique([chartId, position], map: "idx_pillar_chart_position")
  @@map("pillars")
}
```
