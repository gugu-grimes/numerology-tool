# 数据库设计 — 06. 大运流年模块

## 概述

本模块负责起运年龄与大运排列、流年排列、岁运冲合刑害标注的数据持久化。核心数据表包括：`DaYun`（大运表）存储起运年龄、大运顺逆方向与每柱大运天干地支及起止年龄区间；`LiuNian`（流年表）存储流年天干地支、公历年份与虚岁年龄；`DaYunHechong`（岁运冲合刑害表）存储大运与流年对命局的冲合刑害关系及辨病相关度标注。三张表均通过 `chartId` 引用模块 01 的 Chart 实体（排盘数据），并依赖模块 03 的合冲刑害识别逻辑进行岁运冲合刑害分析。

根据架构契约 `architecture.md §③ 数据归属`：**模块 06 拥有 DaYun（大运）、LiuNian（流年）、DaYunHechong（岁运冲合刑害）实体**。

ID 策略遵循 ADR-004（自增整数）；时间存储遵循 ADR-001（UTC）；软删除遵循 ADR-002（`deleted_at` 字段）。

## 表清单

| 表名 | 说明 | 归属模块 |
|------|------|----------|
| DaYun | 大运记录，存储起运年龄、大运顺逆方向与每柱大运天干地支及起止年龄区间 | 本模块（06） |
| LiuNian | 流年记录，存储流年天干地支、公历年份与虚岁年龄 | 本模块（06） |
| DaYunHechong | 岁运冲合刑害记录，存储大运与流年对命局的冲合刑害关系及辨病相关度标注 | 本模块（06） |

## 模块专属表

### DaYun（大运）

存储命盘的大运排列结果，每条记录对应一次排盘的全量大运数据。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Int | PK, AUTOINCREMENT | 大运记录 ID（ADR-004 自增整数） |
| chartId | Int | NOT NULL, FK → Chart.id, UNIQUE | 关联命盘，一对一 |
| startAge | Int | NOT NULL | 起运年龄（岁） |
| direction | String | NOT NULL | 大运顺逆方向：`"顺排"` / `"逆排"` |
| dayunList | Json | NOT NULL | 大运排列列表（见 JSON 结构定义） |
| createdAt | DateTime | NOT NULL, DEFAULT now() | 创建时间 |
| updatedAt | DateTime | NOT NULL, @updatedAt | 更新时间 |
| deletedAt | DateTime? | — | 软删除时间戳（ADR-002） |

**索引**：

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_dayun_chart | chartId | UNIQUE | 每个命盘仅一条大运记录 |

**外键**：

| 字段 | 引用 | 说明 |
|------|------|------|
| chartId | Chart.id | 所属命盘，级联删除 |

## 跨模块数据引用

| 引用数据 | 来源模块 | 引用方式 | 说明 |
|----------|----------|----------|------|
| Chart + Pillar | 01-八字排盘与历法 | 通过 `chartId` 引用 | 读取四柱天干地支数据、性别、出生时间作为起运计算与大运排列的输入 |
| HechongRelation | 03-合冲刑害 | 通过 `chartId` 引用 | 读取合冲刑害识别规则，复用模块 03 的合冲刑害识别逻辑进行岁运冲合刑害分析 |
| YongShenJiXi | 04-辨病与用神 | 通过 `chartId` 引用 | 辨病模块读取用神喜忌数据，用于岁运药效评估（模块 04 引用本模块数据） |

## Prisma Schema

```prisma
/// 大运记录 — 模块 06 创建并拥有
model DaYun {
  id              Int       @id @default(autoincrement())
  chartId         Int       @unique                                    // 关联命盘，一对一
  startAge        Int                                                  // 起运年龄（岁）
  direction       String                                               // 大运顺逆方向："顺排" / "逆排"
  dayunList       Json                                                 // 大运排列列表
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt
  deletedAt       DateTime?                                            // 软删除（ADR-002）

  chart           Chart     @relation(fields: [chartId], references: [id], onDelete: Cascade)

  @@index([chartId], map: "idx_dayun_chart")
  @@map("dayun_records")
}

/// 流年记录 — 模块 06 创建并拥有
model LiuNian {
  id              Int       @id @default(autoincrement())
  chartId         Int       @unique                                    // 关联命盘，一对一
  liunianList     Json                                                 // 流年排列列表
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt
  deletedAt       DateTime?                                            // 软删除（ADR-002）

  chart           Chart     @relation(fields: [chartId], references: [id], onDelete: Cascade)

  @@index([chartId], map: "idx_liunian_chart")
  @@map("liunian_records")
}

/// 岁运冲合刑害记录 — 模块 06 创建并拥有
model DaYunHechong {
  id              Int       @id @default(autoincrement())
  chartId         Int       @unique                                    // 关联命盘，一对一
  dayunHechong    Json                                                 // 大运冲合刑害标注结果
  liunianHechong  Json                                                 // 流年冲合刑害标注结果
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt
  deletedAt       DateTime?                                            // 软删除（ADR-002）

  chart           Chart     @relation(fields: [chartId], references: [id], onDelete: Cascade)

  @@index([chartId], map: "idx_dayun_hechong_chart")
  @@map("dayun_hechong_records")
}
```
