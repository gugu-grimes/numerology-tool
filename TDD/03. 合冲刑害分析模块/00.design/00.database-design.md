# 数据库设计 — 03. 合冲刑害分析模块

## 概述

本模块负责合冲刑害关系的识别、条件判定与辨病评估的数据持久化。核心数据表 `HechongRelation` 存储天干五合、地支六合三合、六冲三刑六害自刑的识别结果与辨病判定，通过 `chartId` 引用模块 01 的 Chart 实体（排盘数据），并依赖模块 02 的五行与十神分析结果进行辨病判定。

根据架构契约 `architecture.md §③ 数据归属`：**模块 03 拥有 CombinationClash（合冲刑害）实体**，包含天干合化、地支合局、冲刑害识别与辨病判定的全部数据。

ID 策略遵循 ADR-004（自增整数）；时间存储遵循 ADR-001（UTC）；软删除遵循 ADR-002（`deleted_at` 字段）。

## 表清单

| 表名 | 说明 | 归属模块 |
|------|------|----------|
| HechongRelation | 合冲刑害关系记录，存储天干五合、地支六合三合、六冲三刑六害自刑的识别结果与辨病判定 | 本模块（03） |

## 模块专属表

### HechongRelation（合冲刑害关系）

存储命盘的合冲刑害关系识别结果与辨病判定，每条记录对应一次排盘的全量合冲刑害分析。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Int | PK, AUTOINCREMENT | 合冲刑害关系 ID（ADR-004 自增整数） |
| chartId | Int | NOT NULL, FK → Chart.id, UNIQUE | 关联命盘，一对一 |
| tianganHe | Json | NOT NULL | 天干五合识别结果（见 JSON 结构定义） |
| dizhiLiuhe | Json | NOT NULL | 地支六合识别结果（见 JSON 结构定义） |
| dizhiSanhe | Json | NOT NULL | 地支三合局识别结果（见 JSON 结构定义） |
| liuchong | Json | NOT NULL | 六冲识别结果（见 JSON 结构定义） |
| sanxing | Json | NOT NULL | 三刑识别结果（见 JSON 结构定义） |
| liuhai | Json | NOT NULL | 六害识别结果（见 JSON 结构定义） |
| ziXing | Json | NOT NULL | 自刑识别结果（见 JSON 结构定义） |
| bingJudgments | Json | NOT NULL | 辨病判定结果（见 JSON 结构定义） |
| createdAt | DateTime | NOT NULL, DEFAULT now() | 创建时间 |
| updatedAt | DateTime | NOT NULL, @updatedAt | 更新时间 |
| deletedAt | DateTime? | — | 软删除时间戳（ADR-002） |

**索引**：

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_hechong_chart | chartId | UNIQUE | 每个命盘仅一条合冲刑害关系记录 |

**外键**：

| 字段 | 引用 | 说明 |
|------|------|------|
| chartId | Chart.id | 所属命盘，级联删除 |

## JSON 结构定义

### tianganHe（天干五合识别结果）

```json
[
  {
    "stem1": "甲",
    "position1": "year",
    "stem2": "己",
    "position2": "month",
    "combination": "甲己合",
    "targetElement": "土",
    "isTransformed": true,
    "transformCondition": "得令且得势",
    "reason": "合化之五行得令且得势，合化成功"
  }
]
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| stem1 | String | 是 | 参与合化的第一个天干 |
| position1 | String | 是 | 第一个天干所在柱位：`"year"` / `"month"` / `"day"` / `"hour"` |
| stem2 | String | 是 | 参与合化的第二个天干 |
| position2 | String | 是 | 第二个天干所在柱位 |
| combination | String | 是 | 五合组合名称（如 `"甲己合"`、`"乙庚合"`） |
| targetElement | String | 是 | 合化目标五行属性（如 `"土"`、`"金"`） |
| isTransformed | Boolean | 是 | 是否合化成功：`true`（合化成功）/ `false`（合而不化） |
| transformCondition | String? | 否 | 合化条件说明（合化成功时描述得令/得势条件；合而不化时为 `null`） |
| reason | String | 是 | 判定原因描述 |

## Prisma Schema

```prisma
/// 合冲刑害关系记录 — 模块 03 创建并拥有
model HechongRelation {
  id              Int       @id @default(autoincrement())
  chartId         Int       @unique                                    // 关联命盘，一对一
  tianganHe       Json                                                 // 天干五合识别结果
  dizhiLiuhe      Json                                                 // 地支六合识别结果
  dizhiSanhe      Json                                                 // 地支三合局识别结果
  liuchong        Json                                                 // 六冲识别结果
  sanxing         Json                                                 // 三刑识别结果
  liuhai          Json                                                 // 六害识别结果
  ziXing          Json                                                 // 自刑识别结果
  bingJudgments   Json                                                 // 辨病判定结果
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt
  deletedAt       DateTime?                                            // 软删除（ADR-002）

  chart           Chart     @relation(fields: [chartId], references: [id], onDelete: Cascade)

  @@index([chartId], map: "idx_hechong_chart")
  @@map("hechong_relations")
}
```
