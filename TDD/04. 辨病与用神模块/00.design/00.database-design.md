# 数据库设计 — 04. 辨病与用神模块

## 概述

本模块负责辨病诊断、用神喜忌推导、断吉凶与岁运药效评估的数据持久化。核心数据表包括 `BingMachine`（病机诊断）、`YongShenJiXi`（用神喜忌）、`JiXiongVerdict`（吉凶论断），三表均通过 `chartId` 引用模块 01 的 Chart 实体（排盘数据），并依赖模块 02（五行与十神）、模块 03（合冲刑害）的分析结果进行辨病与论断计算。

根据架构契约 `architecture.md §③ 数据归属`：**模块 04 拥有 DiseaseDiagnosis（辨病诊断）与 FortuneJudgment（断吉凶）实体**，包含病机识别、用神喜忌推导与六维论断的全部数据。

ID 策略遵循 ADR-004（自增整数）；时间存储遵循 ADR-001（UTC）；软删除遵循 ADR-002（`deleted_at` 字段）。

## 表清单

| 表名 | 说明 | 归属模块 |
|------|------|----------|
| BingMachine | 病机诊断记录，存储八大病机类型的逐项检测结果 | 本模块（04） |
| YongShenJiXi | 用神喜忌记录，存储用神、喜神、忌神、仇神、闲神推导结果与推导逻辑链 | 本模块（04） |
| JiXiongVerdict | 吉凶论断记录，存储六维人生维度论断结果与病机依据 | 本模块（04） |

## JSON 结构定义

### diseases（八大病机检测结果列表）

```json
[
  {
    "type": "dayMasterExcess",
    "name": "身旺无依",
    "position": "日主及参与帮扶的比劫与印星所在柱位",
    "manifestation": "刚愎自用、独断专行、六亲缘薄",
    "severity": "severe",
    "hasRelief": false,
    "diseaseSource": "日主过旺"
  }
]
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | String | 是 | 病机类型枚举：`"dayMasterExcess"` / `"dayMasterWeakness"` / `"wuxingImbalance"` / `"wuxingAbsence"` / `"shishenConflict"` / `"gejuBreak"` / `"heBindYongShen"` / `"temperatureImbalance"` |
| name | String | 是 | 病名（如 `"身旺无依"`、`"身弱无根"`、`"某行偏旺"`、`"缺某行"`、`"某十神克破某十神"`、`"某格被某十神破败"`、`"用神被合绊"`、`"命局过寒无火调候"`） |
| position | String | 是 | 病位（受影响的天干地支所在柱位或命局全局） |
| manifestation | String | 是 | 病象（外在表现描述） |
| severity | String | 是 | 严重程度：`"severe"`（重病）/ `"moderate"`（微恙）/ `"latent"`（潜伏） |
| hasRelief | Boolean | 是 | 是否有化解手段：`true`（有化解）/ `false`（无化解） |
| diseaseSource | String | 是 | 病机来源描述 |

### yongShen（用神详情）

```json
{
  "element": "木",
  "shishen": "比肩",
  "position": ["日干", "年支本气"],
  "derivationLogic": "身弱则印比为用",
  "diseaseSource": "身弱无根",
  "strength": "moderate"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| element | String | 是 | 用神五行属性（`"金"` / `"木"` / `"水"` / `"火"` / `"土"`） |
| shishen | String | 是 | 用神十神属性（如 `"比肩"`、`"正官"`、`"偏财"`） |
| position | Array | 是 | 用神在四柱中的分布位置 |
| derivationLogic | String | 是 | 推导逻辑（从何病推出何药） |
| diseaseSource | String | 是 | 推导所依据的病机 |
| strength | String | 是 | 用神力量：`"strong"`（得力）/ `"moderate"`（适中）/ `"weak"`（无力） |

### xiShen / jiShen / chouShen / xianShen（喜忌仇闲列表）

```json
[
  {
    "element": "水",
    "shishen": "正印",
    "position": ["月干"],
    "derivationLogic": "生助用神木，增强药力",
    "diseaseSource": "身弱无根"
  }
]
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| element | String | 是 | 五行属性 |
| shishen | String | 是 | 十神属性 |
| position | Array | 是 | 在四柱中的分布位置 |
| derivationLogic | String | 是 | 推导逻辑（从何病推出何药或何忌） |
| diseaseSource | String | 是 | 推导所依据的病机 |

### derivationChains（推导逻辑链列表）

```json
[
  {
    "step": 1,
    "disease": "身弱无根",
    "diseaseType": "dayMasterWeakness",
    "yongShenCandidate": "印比",
    "logic": "身弱则印比为用",
    "isFinalYongShen": true
  }
]
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| step | Int | 是 | 推导步骤序号 |
| disease | String | 是 | 病机名称 |
| diseaseType | String | 是 | 病机类型（与 diseases.type 一致） |
| yongShenCandidate | String | 是 | 该病机推导出的用神候选 |
| logic | String | 是 | 推导逻辑描述 |
| isFinalYongShen | Boolean | 是 | 是否为最终确定的用神 |

### personality / career / wealth / marriage / health / education（维度论断）

```json
{
  "conclusion": "性格偏刚毅自主，外向善表达",
  "diseaseBasis": "身旺无依",
  "yongShenLogic": "身旺则官杀财食为用，食伤为用则外向善表达",
  "severity": "moderate",
  "details": "日主得令得地，性格刚毅果断；食伤为用，善表达善交际"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| conclusion | String | 是 | 论断结论 |
| diseaseBasis | String | 是 | 病机依据 |
| yongShenLogic | String | 是 | 用神推导链 |
| severity | String | 是 | 严重程度/吉凶程度：`"severe"` / `"moderate"` / `"mild"` |
| details | String | 是 | 详细论断描述 |
