# 数据库设计 — 08. 命盘历史与比较模块

## 概述

本模块负责命盘历史的持久化保存与双人合盘合婚分析结果的持久化存储。核心数据表为 `SavedChart`（命盘历史记录表）与 `HepanResult`（合盘合婚结果表）。`SavedChart` 存储用户保存的命盘记录及其分析数据快照（四柱概要、五行统计、十神标注、病机清单、用神喜忌、合冲刑害），通过 `chartId` 引用模块 01 的 Chart 实体。`HepanResult` 存储双人命盘比较的分析结果（天干合化、地支合冲刑害、五行互补、冲克叠加、辨病视角结论及合婚分析），通过 `chartId1` 与 `chartId2` 引用两个命盘。

根据架构契约 `architecture.md §③ 数据归属`：**模块 08 拥有 ChartHistory（命盘历史）与 Synastry（合盘/合婚）实体**。

ID 策略遵循 ADR-004（自增整数）；时间存储遵循 ADR-001（UTC）；软删除遵循 ADR-002（`deleted_at` 字段）；分页策略遵循 ADR-005（cursor 分页）。

## 表清单

| 表名 | 说明 | 归属模块 |
|------|------|----------|
| SavedChart | 命盘历史记录，存储用户保存的命盘元数据与分析数据快照 | 本模块（08） |
| HepanResult | 合盘合婚分析结果，存储双人命盘比较的全部分析数据 | 本模块（08） |

## 模块专属表

### SavedChart（命盘历史记录）

存储用户保存的命盘记录，每条记录对应一次命盘保存，包含命盘元数据与分析数据快照。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Int | PK, AUTOINCREMENT | 命盘历史记录 ID（ADR-004 自增整数） |
| chartId | Int | NOT NULL, FK → Chart.id | 关联原始命盘（模块 01 的 Chart） |
| name | String | NOT NULL | 用户指定的命盘名称（同一用户下不可重名） |
| gender | String | NOT NULL | 性别：`"male"` / `"female"`（保存时从 Chart 复制） |
| birthDate | DateTime | NOT NULL | 公历出生日期时间（保存时从 Chart 复制，UTC） |
| pillarSummary | Json | NOT NULL | 四柱概要（天干地支摘要，见 JSON 结构定义） |
| wuxingSnapshot | Json? | — | 五行统计快照（保存时从模块 02 分析结果复制） |
| shishenSnapshot | Json? | — | 十神标注快照（保存时从模块 02 分析结果复制） |
| bingSnapshot | Json? | — | 病机清单快照（保存时从模块 04 分析结果复制） |
| yongshenSnapshot | Json? | — | 用神喜忌快照（保存时从模块 04 分析结果复制） |
| hechongSnapshot | Json? | — | 合冲刑害快照（保存时从模块 03 分析结果复制） |
| createdAt | DateTime | NOT NULL, DEFAULT now() | 创建时间 |
| updatedAt | DateTime | NOT NULL, @updatedAt | 更新时间 |
| deletedAt | DateTime? | — | 软删除时间戳（ADR-002） |

**索引**：

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_saved_chart_created | createdAt | ASC | 命盘列表按保存时间排序，支持 cursor 分页 |
| idx_saved_chart_name | name | NON-UNIQUE | 命盘名称搜索（唯一性由应用层校验） |

**外键**：

| 字段 | 引用 | 说明 |
|------|------|------|
| chartId | Chart.id | 关联原始命盘，Restrict 删除（命盘历史保留独立快照） |

### HepanResult（合盘合婚结果）

存储双人命盘比较分析的全部结果，包含天干合化、地支合冲刑害、五行互补、冲克叠加、辨病视角结论与合婚分析。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Int | PK, AUTOINCREMENT | 合盘结果 ID（ADR-004 自增整数） |
| chartId1 | Int | NOT NULL, FK → Chart.id | 甲方命盘（模块 01 的 Chart） |
| chartId2 | Int | NOT NULL, FK → Chart.id | 乙方命盘（模块 01 的 Chart） |
| analysisType | String | NOT NULL | 分析类型：`"hepan"`（合盘）/ `"hemian"`（合婚） |
| tianganHeResults | Json | NOT NULL | 天干合化结果（见 JSON 结构定义） |
| dizhiHeResults | Json | NOT NULL | 地支六合三合结果（见 JSON 结构定义） |
| dizhiChongResults | Json | NOT NULL | 地支六冲结果（见 JSON 结构定义） |
| dizhiXingHaiResults | Json | NOT NULL | 地支三刑六害结果（见 JSON 结构定义） |
| wuxingComplementary | Json | NOT NULL | 五行互补关系（见 JSON 结构定义） |
| chongkeOverlap | Json | NOT NULL | 冲克叠加关系（见 JSON 结构定义） |
| bingPerspective | Json | NOT NULL | 辨病视角结论（见 JSON 结构定义） |
| marriageAnalysis | Json? | — | 合婚分析（仅 analysisType 为 `"hemian"` 时有值，见 JSON 结构定义） |
| createdAt | DateTime | NOT NULL, DEFAULT now() | 创建时间 |
| updatedAt | DateTime | NOT NULL, @updatedAt | 更新时间 |
| deletedAt | DateTime? | — | 软删除时间戳（ADR-002） |

**索引**：

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_hepan_charts | chartId1, chartId2 | NON-UNIQUE | 按双命盘 ID 查询合盘结果 |
| idx_hepan_created | createdAt | ASC | 按创建时间排序，支持 cursor 分页 |

**外键**：

| 字段 | 引用 | 说明 |
|------|------|------|
| chartId1 | Chart.id | 甲方命盘，Restrict 删除 |
| chartId2 | Chart.id | 乙方命盘，Restrict 删除 |

> 注：`chartId1` 与 `chartId2` 不可相同（PRD 业务规则：两命盘不可为同一命盘），此约束由应用层校验，不在 DB 层强制。

| 引用数据 | 来源模块 | 引用方式 | 说明 |

model SavedChart {
  id               Int       @id @default(autoincrement())
  chartId          Int                                                // 关联原始命盘
  name             String                                             // 用户指定的命盘名称
  gender           String                                             // 性别（保存时从 Chart 复制）

  pillarSummary    Json                                               // 四柱概要：[{position, heavenlyStem, earthlyBranch}]

  shishenSnapshot  Json?                                              // 十神标注快照（保存时从模块 02 复制）

  updatedAt        DateTime  @updatedAt
