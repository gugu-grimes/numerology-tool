# 数据库设计 — 05. 神煞标注模块

## 概述

本模块负责神煞识别与标注、神煞配置管理的数据持久化。核心数据表包括 `ShenshaLabel`（神煞标注记录）与 `ShenshaConfig`（神煞配置），两表通过 `chartId` 引用模块 01 的 Chart 实体，`ShenshaLabel` 同时通过 `schoolId` 引用 `ShenshaConfig` 以标识识别所用的流派规则集。

根据架构契约 `architecture.md §③ 数据归属`：**模块 05 拥有 ShenSha（神煞）实体**，包含神煞识别结果与流派配置的全部数据。

ID 策略遵循 ADR-004（自增整数）；时间存储遵循 ADR-001（UTC）；软删除遵循 ADR-002（`deleted_at` 字段）。

## 表清单

| 表名 | 说明 | 归属模块 |
|------|------|----------|
| ShenshaLabel | 神煞标注记录，存储命盘在指定流派规则集下识别到的全部神煞标注 | 本模块（05） |
| ShenshaConfig | 神煞配置，存储流派规则集及其包含的神煞定义列表 | 本模块（05） |

## 模块专属表

### ShenshaLabel（神煞标注）

存储命盘在指定流派规则集下识别到的全部神煞标注结果。每条记录对应一个命盘在一个流派下的神煞识别结果。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Int | PK, AUTOINCREMENT | 神煞标注 ID（ADR-004 自增整数） |
| chartId | Int | NOT NULL, FK → Chart.id | 关联命盘 |
| schoolId | Int | NOT NULL, FK → ShenshaConfig.id | 关联流派规则集，标识识别所用的规则集 |
| labels | Json | NOT NULL | 识别的神煞标注列表（见 JSON 结构定义） |
| createdAt | DateTime | NOT NULL, DEFAULT now() | 创建时间 |
| updatedAt | DateTime | NOT NULL, @updatedAt | 更新时间 |
| deletedAt | DateTime? | — | 软删除时间戳（ADR-002） |

**索引**：

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_shensha_label_chart_school | chartId, schoolId | UNIQUE | 每个命盘在每个流派下仅一条标注记录 |
| idx_shensha_label_chart | chartId | — | 按命盘查询神煞标注 |

**外键**：

| 字段 | 引用 | 说明 |
|------|------|------|
| chartId | Chart.id | 所属命盘，级联删除 |
| schoolId | ShenshaConfig.id | 所用流派规则集，限制删除 |

### ShenshaConfig（神煞配置）

存储流派规则集及其包含的神煞定义列表。每个流派规则集包含一组神煞定义及识别条件。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | Int | PK, AUTOINCREMENT | 神煞配置 ID（ADR-004 自增整数） |
| name | String | NOT NULL, UNIQUE | 流派名称（如"子平法"、"滴天髓法"） |
| description | String? | — | 流派说明 |
| isDefault | Boolean | NOT NULL, DEFAULT false | 是否为默认生效规则集 |
| definitions | Json | NOT NULL | 神煞定义列表（见 JSON 结构定义） |
| createdAt | DateTime | NOT NULL, DEFAULT now() | 创建时间 |
| updatedAt | DateTime | NOT NULL, @updatedAt | 更新时间 |
| deletedAt | DateTime? | — | 软删除时间戳（ADR-002） |

**索引**：

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|
| idx_shensha_config_name_unique | name | UNIQUE | 流派名称全局唯一 |

**外键**：

无外键。ShenshaConfig 为独立配置表。

## JSON 结构定义

### labels（神煞标注列表）

```json
[
  {
    "name": "天乙贵人",
    "aliases": ["天乙"],
    "pillar": "年柱",
    "position": "年支",
    "ganZhi": "丑",
    "attribute": "吉神",
    "description": "天乙贵人是命中最吉之神，主荣华富贵、逢凶化吉",
    "sourceClassic": "《三命通会》",
    "relatedElements": ["用神所在柱位"]
  }
]
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | String | 是 | 神煞名称（如 `"天乙贵人"`、`"驿马"`、`"桃花"`） |
| aliases | Array\<String\> | 是 | 神煞别名列表 |
| pillar | String | 是 | 所在柱位（`"年柱"` / `"月柱"` / `"日柱"` / `"时柱"`） |
| position | String | 是 | 具体位置描述（如 `"年支"`、`"日干"`） |
| ganZhi | String | 是 | 匹配到的天干或地支（如 `"丑"`、`"甲"`） |
| attribute | String | 是 | 吉凶属性（`"吉神"` / `"凶煞"`） |
| description | String | 是 | 神煞含义与判断要点 |
| sourceClassic | String | 否 | 来源经典文献 |
| relatedElements | Array\<String\> | 否 | 与命盘其他要素的关联描述 |

### definitions（神煞定义列表）

```json
[
  {
    "id": "tianyi-guiren",
    "name": "天乙贵人",
    "aliases": ["天乙"],
    "attribute": "吉神",
    "description": "天乙贵人是命中最吉之神，主荣华富贵、逢凶化吉",
    "sourceClassic": "《三命通会》",
    "isActive": true,
    "matchRules": [
      {
        "type": "dizhi_lookup",
        "description": "以日干查年支与日支",
        "scope": ["年支", "日支"],
        "lookupTable": {
          "甲": ["丑", "未"],
          "乙": ["子", "申"],
          "丙": ["亥", "酉"],
          "丁": ["亥", "酉"],
          "戊": ["丑", "未"],
          "己": ["子", "申"],
          "庚": ["丑", "未"],
          "辛": ["寅", "午"],
          "壬": ["卯", "巳"],
          "癸": ["卯", "巳"]
        }
      }
    ]
  }
]
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String | 是 | 定义唯一标识（流派内唯一，如 `"tianyi-guiren"`） |
| name | String | 是 | 神煞名称 |
| aliases | Array\<String\> | 否 | 神煞别名列表 |
| attribute | String | 是 | 吉凶属性（`"吉神"` / `"凶煞"`） |
| description | String | 是 | 神煞含义与判断要点说明 |
| sourceClassic | String | 否 | 来源经典文献 |
| isActive | Boolean | 是 | 是否启用（`true` 参与识别 / `false` 停用不参与识别） |
| matchRules | Array | 是 | 识别条件规则列表（见下方 matchRules 结构定义） |

### matchRules（识别条件规则）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | String | 是 | 匹配类型：`"tiangan_match"`（天干匹配）/ `"dizhi_lookup"`（地支查表）/ `"combination_match"`（天干地支组合匹配） |
| description | String | 是 | 匹配规则说明 |
| scope | Array\<String\> | 是 | 匹配范围（如 `["年支", "日支"]`、`["四柱天干"]`） |
| lookupTable | Object? | 否 | 查表规则（`"dizhi_lookup"` 类型必填），键为天干，值为匹配地支数组 |
| conditions | Array? | 否 | 匹配条件列表（`"tiangan_match"` / `"combination_match"` 类型使用），每个条件包含天干、地支及匹配位置 |

**matchRules 各类型说明**：

- `"tiangan_match"`：基于天干匹配。`scope` 指定匹配范围（如 `"四柱天干"`），`conditions` 指定具体天干及匹配位置。典型示例：羊刃（以日干查四柱地支见禄后一位）。
- `"dizhi_lookup"`：基于地支查表匹配。以某柱天干查指定范围的地支。`lookupTable` 键为天干，值为需匹配的地支数组。典型示例：天乙贵人（以日干查年支与日支见丑未等）。
- `"combination_match"`：天干地支组合匹配。`conditions` 指定天干地支的组合条件及位置。典型示例：华盖（以年支或日支查三合局末位地支）。

## Prisma Schema

```prisma
/// 神煞标注记录 — 模块 05 创建并拥有
model ShenshaLabel {
  id        Int       @id @default(autoincrement())
  chartId   Int                                        // 关联命盘
  schoolId  Int                                        // 关联流派规则集
  labels    Json                                       // 识别的神煞标注列表
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt
  deletedAt DateTime?                                // 软删除（ADR-002）

  chart     Chart         @relation(fields: [chartId], references: [id], onDelete: Cascade)
  school    ShenshaConfig @relation(fields: [schoolId], references: [id], onDelete: Restrict)

  @@unique([chartId, schoolId], map: "idx_shensha_label_chart_school")
  @@index([chartId], map: "idx_shensha_label_chart")
  @@map("shensha_labels")
}

/// 神煞配置 — 模块 05 创建并拥有
model ShenshaConfig {
  id          Int       @id @default(autoincrement())
  name        String    @unique                             // 流派名称
  description String?                                       // 流派说明
  isDefault   Boolean   @default(false)                     // 是否为默认生效规则集
  definitions Json                                          // 神煞定义列表
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
  deletedAt   DateTime?                                     // 软删除（ADR-002）

  labels      ShenshaLabel[]

  @@index([name], map: "idx_shensha_config_name_unique")
  @@map("shensha_configs")
}
```
