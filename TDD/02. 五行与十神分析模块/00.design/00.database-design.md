# 数据库设计 — 02. 五行与十神分析模块

## 概述

本模块负责五行力量分布统计、日主旺衰判定、十神标注与格局判定四项核心分析的数据持久化。四张数据表分别对应四个子模块的计算结果，均通过 `chartId` 引用模块 01 的 Chart 实体（排盘数据），本身为计算结果的缓存与展示载体。

根据架构契约 `architecture.md §③ 数据归属`：**模块 02 拥有 FiveElementAnalysis（五行分析）与 TenGodAnalysis（十神分析）实体**，其中 FiveElementAnalysis 包含五行力量统计与日主旺衰判定结果，TenGodAnalysis 包含十神标注与格局判定结果。

ID 策略遵循 ADR-004（自增整数）；时间存储遵循 ADR-001（UTC）；软删除遵循 ADR-002（`deleted_at` 字段）。

## 表清单

| 表名 | 说明 | 归属模块 |
|------|------|----------|
| WuxingStat | 五行力量统计记录，存储各五行力量值与缺失/过旺标注 | 本模块（02） |
| DayMasterStrength | 日主旺衰判定记录，存储得令/得地/得助判定与旺衰结论 | 本模块（02） |
| ShishenLabel | 十神标注记录，存储四柱天干与藏干的十神关系标注 | 本模块（02） |
| GejuPattern | 格局判定与喜忌记录，存储格局类型、成败分析与喜忌五行十神 | 本模块（02） |

## 模块专属表

### WuxingStat（五行力量统计）

存储命盘的五行力量分布统计结果，每条记录对应一次排盘的五行分析。

| chartId | Int | NOT NULL, FK → Chart.id, UNIQUE | 关联命盘，一对一 |
| metal | Float | NOT NULL | 金的力量加权值 |
| wood | Float | NOT NULL | 木的力量加权值 |
| water | Float | NOT NULL | 水的力量加权值 |
| fire | Float | NOT NULL | 火的力量加权值 |
| earth | Float | NOT NULL | 土的力量加权值 |
| missing | Json | NOT NULL | 五行缺失列表（如 `["水"]`） |
| dominant | Json | NOT NULL | 五行过旺列表（如 `["火","土"]`） |
| createdAt | DateTime | NOT NULL, DEFAULT now() | 创建时间 |
| updatedAt | DateTime | NOT NULL, @updatedAt | 更新时间 |
| deletedAt | DateTime? | — | 软删除时间戳（ADR-002） |

| 索引名 | 字段 | 类型 | 说明 |
|--------|------|------|------|

### JSON 结构定义

#### missing（五行缺失列表）

```json
["水"]
```

值为缺失的五行名称数组，可能为空数组 `[]`（无缺失）。

#### dominant（五行过旺列表）

```json
["火", "土"]
```

值为过旺的五行名称数组，可能为空数组 `[]`（无过旺）。

#### deLing（得令判定详情）

```json
{
  "result": true,
  "monthBranchElement": "火",
  "relation": "生扶",
  "description": "月令地支五行（火）生扶日主五行（木），日主得令"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| result | Boolean | 是 | 是否得令 |
| monthBranchElement | String | 是 | 月令地支五行属性 |
| relation | String | 是 | 月令与日主的关系：`"生扶"` / `"克泄"` / `"同类"` |
| description | String | 是 | 判定描述文字 |

#### deDi（得地判定详情）

```json
{
  "result": true,
  "branches": [
    { "position": "month", "branch": "午", "element": "火", "isRoot": true }
  ],
  "description": "日主在地支有根，得地"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| result | Boolean | 是 | 是否得地 |
| branches | Array | 是 | 有根的地支列表 |
| branches[].position | String | 是 | 柱位：`"year"` / `"month"` / `"day"` / `"hour"` |
| branches[].branch | String | 是 | 地支名称 |
| branches[].element | String | 是 | 地支五行属性 |
| branches[].isRoot | Boolean | 是 | 是否为日主之根 |
| description | String | 是 | 判定描述文字 |

#### deZhu（得助判定详情）

```json
{
  "result": false,
  "helpingStems": [],
  "description": "天干无同类或生扶日主者，不得助"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| result | Boolean | 是 | 是否得助 |
| helpingStems | Array | 是 | 提供帮助的天干列表 |
| helpingStems[].position | String | 是 | 柱位：`"year"` / `"month"` / `"hour"` |
| helpingStems[].stem | String | 是 | 天干名称 |
| helpingStems[].relation | String | 是 | 与日主的关系：`"同类"` / `"生扶"` |
| description | String | 是 | 判定描述文字 |

#### stemLabels（四柱天干十神标注）

```json
{
  "year": { "stem": "庚", "god": "偏官" },
  "month": { "stem": "壬", "god": "偏印" },
  "hour": { "stem": "戊", "god": "食神" }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| [position] | Object | 是 | 各柱位天干十神标注 |
| [position].stem | String | 是 | 天干名称 |
| [position].god | String | 是 | 十神名称（日柱天干为日主，标注为 `"日主"`） |

#### hiddenStemLabels（地支藏干十神标注）

```json
{
  "year": {
    "branch": "午",
    "mainQi": { "stem": "丁", "god": "伤官" },
    "middleQi": { "stem": "己", "god": "正财" },
    "residualQi": null
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| [position] | Object | 是 | 各柱位地支藏干十神标注 |
| [position].branch | String | 是 | 地支名称 |
| [position].mainQi | Object | 是 | 本气藏干十神标注 |
| [position].mainQi.stem | String | 是 | 本气天干名称 |
| [position].mainQi.god | String | 是 | 本气十神名称 |
| [position].middleQi | Object? | 否 | 中气藏干十神标注（无中气时为 `null`） |
| [position].residualQi | Object? | 否 | 余气藏干十神标注（无余气时为 `null`） |
