# 用神喜忌推导

## Part 1 业务流程

### 1.1 用神推导主流程

```mermaid
graph TD
    A["系统接收识病诊断输出的病机清单"] --> B["系统读取日主旺衰判定结果"]

    C -->|"是"| D["系统以克泄耗日主的五行与十神为用神候选"]
    C -->|"否"| E{"日主是否身弱或从弱?"}
    E -->|"是"| F["系统以生扶日主的五行与十神为用神候选"]
    E -->|"否"| G{"命局是否存在五行偏枯或缺漏?"}
    G -->|"是"| H["系统以调候补救所缺所偏之五行为用神候选"]
    G -->|"否"| I{"命局是否存在十神交战?"}
    I -->|"是"| J["系统以化解交战的十神为用神候选"]
    I -->|"否"| K{"命局是否存在格局破败?"}
    K -->|"是"| L["系统以修补格局的十神为用神候选"]
    K -->|"否"| M{"命局是否存在合绊用神?"}
    M -->|"是"| N["系统以冲开合绊、释放用神的五行或十神为用神候选"]
    M -->|"否"| O{"命局是否存在寒暖湿燥失衡?"}
    O -->|"是"| P["系统以调候所需之五行为用神候选"]
    O -->|"否"| Q["系统以扶抑适中的五行与十神为用神"]
    D --> R["系统从候选中确定最有力的一位用神"]
    F --> R
    H --> R
    J --> R
    L --> R
    N --> R
    P --> R
    Q --> R
    R --> S["系统标注用神的推导逻辑（从何病推出何药）"]
    S --> T["系统输出用神及其推导链"]
```

### 1.2 喜忌仇闲推导流程

```mermaid
graph TD
    A["系统完成用神推导"] --> B["系统根据用神推导喜神"]
    B --> C["系统识别生助用神、增强药力的五行与十神为喜神"]
    C --> D["系统根据病机推导忌神"]

    E --> F["系统根据忌神推导仇神"]
    F --> G["系统识别生助忌神、间接加重病情的五行与十神为仇神"]
    G --> H["系统推导闲神"]
    H --> I["系统识别对命局影响不大的五行与十神为闲神"]
    I --> J["系统为每项喜忌标注推导逻辑（从何病推出何药或何忌）"]
    J --> K["系统输出完整的喜忌五行与十神列表"]
    K --> L["系统将喜忌推导结果输出至断吉凶与岁运药效评估子模块"]
```

## Part 2 关键页面功能列表

### 页面 / 功能 1: 用神推导页

- **URL / 路径（业务命名）**: 用神推导页
- **目标用户**: 命理学习者、命理从业者、普通用户
- **核心功能**:
  - 查看命局用神及其五行与十神属性
  - 查看用神的推导逻辑链（从何病推出何药）
  - 查看用神在四柱中的分布位置

### 页面 / 功能 2: 喜忌仇闲列表页

- **URL / 路径（业务命名）**: 喜忌仇闲列表页
- **目标用户**: 命理学习者、命理从业者、普通用户
- **核心功能**:
  - 查看喜神列表及其五行与十神属性
  - 查看忌神列表及其五行与十神属性
  - 查看仇神列表及其五行与十神属性
  - 查看闲神列表及其五行与十神属性
  - 查看每项喜忌的推导逻辑（从何病推出何药或何忌）

### 页面 / 功能 3: 推导逻辑链详情页

- **URL / 路径（业务命名）**: 推导逻辑链详情页
- **目标用户**: 命理学习者、命理从业者、普通用户
- **核心功能**:
  - 查看从病机到用神的完整推导链
  - 查看从用神到喜神的辅助推导逻辑
  - 查看从病机到忌神的对抗推导逻辑
  - 查看从忌神到仇神的间接推导逻辑
  - 查看闲神的判定依据
