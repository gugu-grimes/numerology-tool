# 天干合化

## Part 1 业务流程

### 1.1 天干合化识别与判定主流程

```mermaid
graph TD
    A["用户查看命盘的天干合化分析"] --> B["系统读取四柱天干数据"]
    B --> C["系统识别四柱间所有天干五合组合"]
    C --> D["系统对每组五合组合判断合化是否成功"]
    D --> E["系统标注合化成功的天干组合及化出五行"]
    D --> F["系统标注合而不化的天干组合"]
    E --> G["页面展示天干合化结果清单"]
    F --> G
```

### 1.2 合化成功条件判定流程

```mermaid
graph TD
    A["系统对每组天干五合组合进行条件判定"] --> B{"合化之五行是否得令?"}
    B -->|"是"| C{"合化之五行是否得势?"}
    B -->|"否"| D["系统判定合而不化"]
    C -->|"是"| E["系统判定合化成功，标注化出五行"]
    C -->|"否"| D
    D --> F["页面展示合而不化状态及原因"]
    E --> G["页面展示合化成功结果"]
```

## Part 2 关键页面功能列表

### 页面 / 功能 1: 天干合化结果页

- **URL / 路径（业务命名）**: 天干合化结果页
- **目标用户**: 命理学习者、命理从业者、普通用户
- **核心功能**:
  - 查看四柱间天干五合组合列表
  - 查看每组五合的合化状态（合化成功或合而不化）
  - 查看合化成功组合的化出五行属性
  - 查看合而不化组合的未化原因
